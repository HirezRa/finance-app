# תיקוני Encoding — טקסט עברי

## תאריך: 2026-04-29

## סיכום

- **נסרקו:** `frontend/src/`, `backend/src/` (חיפוש דפוס `×` ומחרוזות mojibake).
- **ב־`backend/src`:** לא נמצאו מחרוזות משובשות.
- **ב־`frontend/src`:** נמצאו **3 קבצים** עם טקסט עברי שגוי (UTF-8 שפורש כ־Latin-1/CP1252).
- **תיקון:** שימוש ב־**[ftfy](https://github.com/rspeer/python-ftfy)** (`ftfy.fix_text` על שורות שמכילות `×`) + **תיקון ידני** ל־3 שורות שלא פוענחו אוטומטית (תווי מרכאות / NBSP).

## קבצים שתוקנו

- כל הודעות ה־toast, כותרות, טקסטי UI, ומחרוזות שגיאה — פוענחו מ־mojibake לעברית תקינה.

### `frontend/src/services/api.ts`

- הערות JSDoc בעברית — תוקנו (2 שורות).

### `frontend/src/pages/SettingsPage.tsx`

- תוויות טאבים, כותרות, הודעות, תיאורי כרטיסים, דיאלוגים — פוענחו אוטומטית.
- **תיקון ידני נוסף:**
  - **שורה ~168:** `×›×"×"` → `כהה` (תצוגה כהה).
  - **שורה ~429:** מחרוזת שבורה → `נקה` (כפתור ניקוי לוגים).
  - **שורה ~1061:** `כ"הכנסה"` (ציטוט קטגוריה) — נכתב עם **גרשיים עבריים** `כ״הכנסה״`.

## מניעה לעתיד

1. לשמור קבצי מקור כ־**UTF-8** (ללא BOM), עם סיומות שורה עקביות.
2. נוסף קובץ **`.editorconfig`** בשורש הפרויקט עם `charset = utf-8`.
3. ב־Cursor/VS Code: **Files: Encoding** = UTF-8.
4. אם מופיע שוב דפוס `×` ברצף במחרוזות — לבדוק encoding לפני commit (או להריץ `ftfy.fix_text` על שורות חשודות).

## כלי עזר (אופציונלי)

```bash
pip install ftfy
python -c "import ftfy; print(ftfy.fix_text(open('path/to/file.tsx', encoding='utf-8').read()))"
```
