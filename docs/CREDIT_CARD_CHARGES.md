# מניעת כפילות חיובי אשראי

## הבעיה

כשמחברים גם חשבון בנק וגם כרטיס אשראי:

- חברת האשראי מדווחת על כל עסקה בנפרד.
- הבנק מדווח על חיוב כולל (למשל "ישראכרט -11,223₪", "מקס איט", "כרטיסי אשראי").

אם שני המקורות נספרים באותו תקציב — נוצרת **כפילות**.

## הפתרון

עסקאות שמזוהות כ**חיוב אשראי אגרגטיבי מחשבון בנק** מסומנות אוטומטית כ־`isExcludedFromCashFlow = true`, כך שלא ייכנסו לחישוב תקציב/תזרים (בהתאם לשאילתות המערכת).

ניתן גם לסמן או לבטל סימון ידנית מהמסך **עסקאות** (תפריט פעולות).

## זיהוי אוטומטי

מילות מפתח בתיאור (עברית ואנגלית, ללא רגישות לרישיות), לדוגמה:

- ישראכרט / isracard  
- כרטיסי אשראי  
- מקס איט / max it  
- ויזה כאל / visa cal  
- לאומי קארד / leumi card  
- אמריקן אקספרס / american express  
- פרימיום אקספרס  
- דיינרס / diners  

**תנאים:** חשבון מסוג `BANK`, סכום שלילי (חיוב), והגדרת המשתמש להחרגה פעילה.

עסקאות קיימות: ניתן להריץ סקריפט חד־פעמי (בקונטיינר backend):

```bash
npm run fix:credit-charges
# או: npx ts-node prisma/fix-credit-charges.ts
```

## הגדרות משתמש

בהגדרות → התראות:

- **"הסתר חיובי אשראי מחשבון בנק"** — `excludeCreditCardChargesFromBudget`  
  - **מופעל (ברירת מחדל):** זיהוי אוטומטי בסנכרון.  
  - **כבוי:** לא מסמנים אוטומטית (ניתן עדיין לסמן ידנית).

## הערה שנשמרת

בעת זיהוי אוטומטי נשמרת הערה בעברית כדי להבהיר את המקור, לדוגמה:  
`חיוב אשראי - לא נספר בתקציב (נספר דרך חברת האשראי)`.
